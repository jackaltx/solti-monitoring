# Loki Role Testing Guide

## Quick Test Commands

### 1. Deploy Service

```bash
cd mylab
./manage-svc.sh loki deploy
```

**Expected Results:**
- Loki package installed from Grafana repository
- Service `loki.service` enabled and started
- Configuration file created at `/etc/loki/config.yml`
- Storage directories created (local or S3 configured)
- HTTP API accessible on port 3100

### 2. Verify Deployment

```bash
# Automated verification
cd mylab
./svc-exec.sh loki verify

# Manual checks
systemctl status loki
loki --version
curl -I http://localhost:3100/ready
```

**Expected Results:**
- Service status: `active (running)`
- Service enabled: `enabled`
- HTTP ready endpoint returns 200 OK
- No errors in logs

### 3. Check Loki Health

```bash
# Check ready endpoint
curl http://localhost:3100/ready

# Check metrics endpoint
curl http://localhost:3100/metrics | head -20

# List configured labels
curl http://localhost:3100/loki/api/v1/labels

# Query logs (if any data exists)
curl -G "http://localhost:3100/loki/api/v1/query" \
  --data-urlencode 'query={job="varlogs"}' \
  --data-urlencode 'limit=10'
```

**Expected Results:**
- `/ready` returns `ready`
- `/metrics` shows Prometheus-format metrics
- `/labels` returns configured label names (may be empty if no data yet)
- Queries execute successfully (may return no results if fresh install)

## Configuration Architecture

### Configuration Files

```
/etc/loki/
└── config.yml                      # Main configuration (generated by role)

/var/lib/loki/                      # Data directory (local storage)
├── chunks/                         # Compressed log chunks
├── rules/                          # Alert/recording rules
├── wal/                            # Write-ahead log
└── compactor/                      # Compaction metadata
```

### Loki Configuration Structure

**Single YAML file**: `/etc/loki/config.yml`

```yaml
auth_enabled: false

server:
  http_listen_port: 3100

common:
  path_prefix: /var/lib/loki
  storage:
    filesystem:              # OR s3 if S3 enabled
      chunks_directory: /var/lib/loki/chunks
      rules_directory: /var/lib/loki/rules
  replication_factor: 1

schema_config:
  configs:
    - from: 2024-01-01
      store: tsdb
      object_store: filesystem  # OR s3
      schema: v13
      index:
        prefix: index_
        period: 24h

storage_config:
  tsdb_shipper:
    active_index_directory: /var/lib/loki/tsdb-index
    cache_location: /var/lib/loki/tsdb-cache

limits_config:
  retention_period: 744h  # 31 days
```

### How Role Variables Map to Configuration

```yaml
# In inventory:
loki_local_storage: true
  ↓
# Generates in config.yml:
common:
  storage:
    filesystem:
      chunks_directory: /var/lib/loki/chunks
      rules_directory: /var/lib/loki/rules
schema_config:
  configs:
    - object_store: filesystem

---

loki_local_storage: false
loki_endpoint: "minio.example.com:9000"
loki_s3_bucket: "loki-logs"
loki_key_id: "ACCESS_KEY"
loki_access_key: "SECRET_KEY"
  ↓
# Generates in config.yml:
common:
  storage:
    s3:
      endpoint: minio.example.com:9000
      bucketnames: loki-logs
      access_key_id: ACCESS_KEY
      secret_access_key: SECRET_KEY
      s3forcepathstyle: true
schema_config:
  configs:
    - object_store: s3
```

### Key Configuration Variables

**Core Variables:**
```yaml
loki_state: present              # Install/remove control
loki_local_storage: true         # Local filesystem vs S3
```

**S3 Storage (when loki_local_storage: false):**
```yaml
loki_endpoint: ""                # S3 endpoint (host:port)
loki_s3_bucket: ""               # S3 bucket name
loki_key_id: ""                  # S3 access key ID
loki_access_key: ""              # S3 secret access key
```

**Advanced:**
```yaml
loki_force_reload: false         # Force package reinstall
loki_delete_config: false        # Remove config on uninstall
loki_delete_data: false          # Remove data on uninstall
```

## Molecule Testing Scenarios

### Proxmox Scenario

```bash
cd solti-monitoring/roles/loki
molecule test -s proxmox
```

**Test Sequence:**
1. `destroy` - Remove any existing test VM
2. `create` - Clone Proxmox template, assign IP
3. `prepare` - Install dependencies, configure SSH
4. `converge` - Run loki role
5. `verify` - Check service status, API endpoints, storage configured
6. `destroy` - Cleanup test VM

**Environment Variables Required:**
```bash
export PROXMOX_VMID=9000          # Unique VM ID
export PROXMOX_TEMPLATE=rocky9    # rocky9, debian12, ubuntu24
export MOLECULE_IP=10.0.50.100    # Static IP for test VM
```

**What Gets Verified** ([molecule/proxmox/verify.yml](molecule/proxmox/verify.yml)):
- [ ] Loki service running and enabled
- [ ] HTTP API accessible on port 3100
- [ ] `/ready` endpoint returns healthy status
- [ ] Configuration file exists with correct permissions
- [ ] Storage directories created (filesystem or S3 configured)
- [ ] No errors in service logs
- [ ] Test report generated in `verify_output/<distro>/`

## Storage Backends

### Local Filesystem Storage (Default)

```yaml
# In inventory
loki_local_storage: true
```

**Creates:**
- `/var/lib/loki/chunks/` - Compressed log chunks
- `/var/lib/loki/rules/` - Alert rules
- `/var/lib/loki/wal/` - Write-ahead log
- `/var/lib/loki/tsdb-index/` - TSDB index
- `/var/lib/loki/tsdb-cache/` - TSDB cache

**Verification:**
```bash
# Check directories created
ls -la /var/lib/loki/

# Check permissions
stat /var/lib/loki/chunks
# Should be owned by loki:loki

# Write test log
curl -XPOST "http://localhost:3100/loki/api/v1/push" \
  -H "Content-Type: application/json" \
  -d '{"streams":[{"stream":{"job":"test"},"values":[["'$(date +%s)'000000000","test message"]]}]}'

# Check chunk files created
ls -lh /var/lib/loki/chunks/
```

**Best For:**
- Single-node deployments
- Development/testing
- Small-scale production (< 100GB/day)
- Fastest query performance

### S3-Compatible Object Storage

```yaml
# In inventory
loki_local_storage: false
loki_endpoint: "minio.example.com:9000"
loki_s3_bucket: "loki-logs"
loki_key_id: "ACCESSKEY123"
loki_access_key: "secretkey456"
```

**Generates:**
```yaml
# In /etc/loki/config.yml
common:
  storage:
    s3:
      endpoint: minio.example.com:9000
      bucketnames: loki-logs
      access_key_id: ACCESSKEY123
      secret_access_key: secretkey456
      s3forcepathstyle: true
      insecure: false
```

**Verification:**
```bash
# Check config has S3 settings
sudo grep -A 10 's3:' /etc/loki/config.yml

# Write test log
curl -XPOST "http://localhost:3100/loki/api/v1/push" \
  -H "Content-Type: application/json" \
  -d '{"streams":[{"stream":{"job":"test"},"values":[["'$(date +%s)'000000000","s3 test message"]]}]}'

# Check S3 bucket for data (after compaction)
aws s3 ls s3://loki-logs/fake/
# Or with MinIO client:
mc ls minio/loki-logs/fake/
```

**Best For:**
- Multi-node deployments
- Long-term retention
- Large-scale production (> 100GB/day)
- Disaster recovery (data in object storage)

**Supported S3 Services:**
- AWS S3
- MinIO
- QNAP QuObject
- Ceph RGW
- Any S3-compatible storage

## LogQL Query Testing

### Basic Queries

```bash
# Get all logs with job label
curl -G "http://localhost:3100/loki/api/v1/query" \
  --data-urlencode 'query={job="varlogs"}' \
  --data-urlencode 'limit=10'

# Filter by hostname
curl -G "http://localhost:3100/loki/api/v1/query" \
  --data-urlencode 'query={hostname="fleur"}' \
  --data-urlencode 'limit=10'

# Search for specific text
curl -G "http://localhost:3100/loki/api/v1/query" \
  --data-urlencode 'query={job="varlogs"} |= "error"' \
  --data-urlencode 'limit=50'
```

### Range Queries

```bash
# Get logs from last hour
curl -G "http://localhost:3100/loki/api/v1/query_range" \
  --data-urlencode 'query={hostname="fleur"}' \
  --data-urlencode "start=$(date -d '1 hour ago' +%s)000000000" \
  --data-urlencode "end=$(date +%s)000000000" \
  --data-urlencode 'limit=100'
```

### Aggregation Queries

```bash
# Count logs by hostname
curl -G "http://localhost:3100/loki/api/v1/query" \
  --data-urlencode 'query=sum by(hostname) (count_over_time({job="varlogs"}[24h]))'

# Rate of error logs
curl -G "http://localhost:3100/loki/api/v1/query" \
  --data-urlencode 'query=rate({job="varlogs"} |= "error" [5m])'
```

### Label Queries

```bash
# List all label names
curl http://localhost:3100/loki/api/v1/labels

# List values for specific label
curl http://localhost:3100/loki/api/v1/label/hostname/values
```

## Verification Checklist

After deployment, verify:

- [ ] Service running: `systemctl is-active loki`
- [ ] Service enabled: `systemctl is-enabled loki`
- [ ] Ready endpoint healthy: `curl http://localhost:3100/ready`
- [ ] Config file exists: `/etc/loki/config.yml`

## Common Errors

### "permission denied" on /var/lib/loki

**Cause:** Loki user lacks write access to data directory

**Fix:**
```bash
# Fix permissions
sudo chown -R loki:loki /var/lib/loki
sudo chmod 755 /var/lib/loki

# Restart
sudo systemctl restart loki
```

### "connection refused" to S3 endpoint

**Cause:** S3 endpoint not accessible or incorrect

**Fix:**
```bash
# Test S3 endpoint
curl -I http://minio.example.com:9000

# Check config
sudo grep endpoint /etc/loki/config.yml

# Update in inventory:
loki_endpoint: "correct-host:9000"

# Redeploy
cd mylab
./manage-svc.sh loki deploy
```

### "access denied" to S3 bucket

**Cause:** Invalid S3 credentials or bucket permissions

**Fix:**
```bash
# Test credentials
aws s3 ls s3://loki-logs/ \
  --endpoint-url http://minio.example.com:9000

# Update credentials in inventory:
loki_key_id: "correct-access-key"
loki_access_key: "correct-secret-key"

# Redeploy
cd mylab
./manage-svc.sh loki deploy
```

### "too many outstanding requests"

**Cause:** Query load exceeding limits

**Fix:**
```bash
# Edit /etc/loki/config.yml
limits_config:
  max_outstanding_requests_per_tenant: 2048  # Increase from default

# Restart
sudo systemctl restart loki
```

### High memory usage

**Cause:** Large queries, many streams, or retention too long

**Fix:**
```bash
# Check active streams
curl http://localhost:3100/metrics | grep loki_ingester_streams

# Reduce retention in config:
limits_config:
  retention_period: 168h  # 7 days instead of 31

# Restart
sudo systemctl restart loki

# Or add more memory to server
```

### Logs not appearing in queries

**Cause:** Multiple possible issues

**Diagnosis:**
```bash
# 1. Check logs being pushed
curl http://localhost:3100/metrics | grep loki_distributor_bytes_received_total
# Should be increasing

# 2. Check ingester is flushing
curl http://localhost:3100/metrics | grep loki_ingester_chunks_flushed_total
# Should be increasing

# 3. Check time range
# Logs may be outside query time range - use wider range

# 4. Check labels match exactly
curl http://localhost:3100/loki/api/v1/labels
# Verify labels exist
```

## Notes

- **Single binary**: Loki runs all components in one process (ingester, distributor, querier, compactor)
- **No index on content**: Only indexes labels, not log content (keeps storage costs low)
- **Chunk storage**: Logs compressed into chunks, stored in filesystem or S3
- **Compaction**: Automatically runs to optimize storage
- **Retention**: Set via `retention_period`, enforced by compactor
- **S3 path style**: `s3forcepathstyle: true` required for MinIO and most S3-compatible storage
- **Local vs S3**: Local faster for queries, S3 better for scale and durability
- **Schema v13**: Current schema version (TSDB), uses less storage than older schemas
- **Write-ahead log (WAL)**: Protects against data loss during ingester crashes
- **Multi-tenancy**: Can be enabled with `auth_enabled: true` (requires tenant ID in headers)
